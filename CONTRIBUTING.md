# Contributing

## Workflow
1. Create a feature branch from `develop`.
2. Make focused, well-scoped changes.
3. Add or update tests when applicable.
4. Ensure `npm test` passes locally.
5. Open a PR against `develop`.

## Coding Guidelines
- Keep APIs RESTful and consistent.
- Prefer centralized validation and error handling.
- Log security and financial actions.
- Avoid introducing breaking changes without versioning.
